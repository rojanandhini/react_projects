What is `NPM`?

Node Package Manager, is the default package manager for Node.js, used to install, share, and manage dependencies, libraries, and tools in JavaScript projects. It consists of a command-line interface (CLI) and an online registry with millions of packages.

What is `Parcel/vite`? Why do we need it?

Vite and Parcel are modern frontend build tools that prioritize speed and developer experience over legacy tools like Webpack. Vite offers extremely fast, on-demand serving using native ES modules for rapid development. Parcel excels with zero-configuration simplicity, acting as a "plug-and-play" bundler for smaller to medium projects.

What is `.parcel-cache`?

The .parcel-cache directory is a temporary cache folder used by the Parcel web application bundler to store information about your project's assets, dependencies, and transformations. This allows Parcel to significantly speed up subsequent build times by only rebuilding files that have changed.

What is `npx`?

npx (Node Package Executor) is a CLI tool included with npm v5.2.0+ that executes Node.js binaries from the registry without needing global installation.

What is the difference between `dependencies` vs `devDependencies`?

The core difference is that dependencies are required for your application to run in production, while devDependencies are only needed for local development and testing. 

What is Tree Shaking?

Tree shaking is a performance optimization technique used in modern JavaScript bundlers (like Webpack or Rollup) to remove unused code—or "dead code"—from the final bundle. It relies on the static structure of ES6 module syntax (import and export) to determine which code is actually executed.

What is Hot Module Replacement?

Hot Module Replacement (HMR) is a development technique that updates, adds, or removes JavaScript modules in a running application without a full page reload. By instantly applying changes to CSS/JS, it drastically speeds up development, preserves application state (like form inputs or modal states), and offers a faster feedback loop than traditional live-reloading.

What is the difference between `package.json` and `package-lock.json`?

package.json is a manifest file that defines a Node.js project's metadata and high-level dependencies using version ranges, while package-lock.json is an auto-generated lockfile that records the exact version of every installed package and its sub-dependencies to ensure consistent and reproducible installs across different environments.

Why should I not modify `package-lock.json`?

We should not modify package-lock.json manually because it is a machine-generated file that guarantees a consistent and reproducible dependency tree across different environments. Modifying it manually can lead to unexpected behavior and synchronization issues with our package.json file.

What is the `dist` folder?

The dist folder (short for distribution) is a directory in software development projects that contains the final, optimized, production-ready files generated by a build process. It holds the processed code that is ready to be deployed to a web server or distributed to end-users, without the need for them to compile or minify the source code themselves.

What is `browserlists`?

Browserslist is an open-source library and configuration standard used in web development to define which browsers and Node.js versions a project should support. 
It acts as a single, centralized source of truth for browser compatibility, allowing various frontend tools (like Babel, Autoprefixer, ESLint, and Webpack) to share this information and automatically tailor their output.
